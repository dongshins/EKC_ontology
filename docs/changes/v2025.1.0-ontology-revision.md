## Versioning metadata policy

The following note clarifies how ontology-level and release-level versioning are distinguished in the present repository structure.

During the final review of `v2025.1.0`, the relationship between `dcterms:replaces` and `owl:priorVersion` was reconsidered in order to determine whether the ontology header should be revised again and republished as `ekc_2025_1_1.ttl`.

The conclusion was to retain the current metadata structure.

In the present repository policy, `http://dh.aks.ac.kr/ontologies/ekc-2025` is treated as the ontology-level version IRI for the EKC 2025 line, while `v2025.1.0` serves as the release-level designation expressed in `owl:versionInfo`, release metadata, and related repository documentation.

Accordingly:

- `dcterms:replaces` continues to indicate that the current EKC 2025 ontology line supersedes `ekc-2022-alpha2507`;
- `owl:priorVersion` continues to identify `ekc-2022-alpha2507` as the ontology-level predecessor of the current `ekc-2025` line;
- `v2025.0.0` and `v2025.1.0` are not modeled here as separate ontology version IRIs linked by `owl:priorVersion`.

For this reason, no additional ontology file such as `ekc_2025_1_1.ttl` is issued solely to reinterpret these two properties.

This decision should be read as a clarification of repository versioning policy, not as a correction of a defective ontology header. Under the present model, the existing values remain appropriate because the EKC 2025 ontology line is being maintained as a single ontology-level version line with release-level refinements documented through `owl:versionInfo`, GitHub release records, and accompanying change documentation.

This policy also provides the basis for the wording adopted in the `README.md`, `CHANGELOG.md`, and release note associated with `v2025.1.0`.
